Gentoo LinuxのEee PC 900へのインストール: Xorgのインストール

(工事中 90%)

カーネルの設定

evdev

Xorgはインプット用のドライバとしてevdevを使用する。 以前はマウスやキーボード等に様々なドライバが使われていた。 evdevに統一されてずいぶん楽になった。

evdevが有効になっていることを確認する。

% cd /usr/src/linux
% sudo make menuconfig
Device Drivers --->
Input device support --->
<*> Event interface

tmuxを使ってペインを分割している場合、ターミナルの大きさが19行に満たない。 この場合Ctrl-b jで上のペインを大きくしてそこでmenuconfigを行えば良い。 今回はすでにevdevは有効だった。

あるいは以下のようにも確認できる。 こっちのほうがスマートだ。

% zcat /proc/config.gz | grep -i evdev
CONFIG_INPUT_EVDEV=y

Kernel modesetting(KMS)

KMSが使われるようになってXのインストールは格段に簡単で安心できるようになった。 CUIの時点ですでにGUI用のモニタの設定ができているので、 Xを入れる前にドライバが問題なく働いていることがわかる。

% sudo make menuconfig
Device Drivers --->
Graphics support --->
Support for frame buffer devices --->
(すべてのドライバを無効にする)
Console display driver support --->
<*> Framebuffer Console Support

これもそのままの設定で問題ない。

ビデオカードの種類を調べる。

% /usr/sbin/lspci | grep -i graphics
00:02.0 VGA compatible controller: Intel Corporation Mobile 915GM/GMS/901GML Express Graphics Controller (rev 04)
00:02.1 Display controller: Intel Corporation Monile 915GM/GMS/910GML Express Graphics Controller (rev 04)

Intelのカードであることがわかる。

カーネルの設定を確認する。

sudo make menuconfig
Device Drivers --->
Graphics support --->
/dev/agpgart (AGP Support) --->
<*> Intel 440LX/BX/GX, I8xx and E7x05 chi...
Direct Rendering Manager (XFree86 4.1.0 and higher ...) --->
<*> Intel 8xx/9xx/G3x/G4x/HD Graphics
[*] Enable modesetting on intel by default

そのままの設定で問題ない。

make.confの設定

make.confに2つの環境変数を設定する。 ついでにここでシステムのデフォルトの言語を日本語に設定してしまおう。 VIDEO_CARDS, INPUT_DEVICES, LINGUASを設定する。

% sudoedit /etc/portage/make.conf
INPUT_DEVICES="evdev"
VIDEO_CARDS="intel"
LINGUAS="ja"

Xorgのインストール

まずは環境変数が正しく設定されていることを確認する。

emerge --ask --verbose --pretend x11-base/xorg-drivers

表示が流れて消えていってしまった場合、tmuxを使っているのならば、 Ctrl-b [としCtrl-uで上を見ていけば良い。 下にもどるにはCtrl-dを使う。 より細かく移動したければCtrl-y, Ctrl-eを使う。 コマンド入力にもどるにはESCを押す。

見ていくと、CPU_FLAGS_X86にsse2を入れておくと良さそうなことに気づく。

% sudoedit /etc/portage/make.conf
...
CPU_FLAGS_X86="sse2"

またついでにUSEにclangを入れておこう。

% sudoedit /etc/portage/make.conf
...
USE="...
...
zsh-complition clang"
...

Xorgをインストールする。

sudo emerge --ask --verbose x11-base/xorg-server

USE等の変数をざっとチェックする。 72個のパッケージがインストールされるので相当時間がかかるはずだ。

現在llvmのインストール中。 これに非常に時間がかかる様子。 次回はllvmを別にインストールしてからXorgをインストールしたほうが良いかもしれない。

最終的に8時間半かかった。

% startx

xtermがないためにエラーとなったようだ。 xtermをインストールする。

% sudo emerge -av xterm

20分ほどかかる。

% startx

twmというウィンドウマネージャが立ち上がる。 1番大きいターミナルでexitコマンドを入力する。

成功

これでXorgのインストールに成功した。
