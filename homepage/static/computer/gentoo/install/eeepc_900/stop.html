Gentoo LinuxのEee PC 900へのインストール: 中断と再開

(工事中 90%)

はじめに

インストールを中断せざるを得ないことがある。 前回の段階以降であればこのページで説明する方法で、 インストールを中断・再開することができる。

インストールの中断

exitしてshutdownする。

# exit
# shutdown -h now

Eee PC 900の本体をおかたづけする。

インストールの再開

Eee PC 900本体の準備

  • 必要な機器の用意
    • Eee PC本体
    • 電源
    • HHKBキーボード
    • USB CD(DVD)-ROMドライブ
    • 各種ケーブル
  • Eee PC 900を電源につなぐ
  • キーボードを接続する
  • CD(DVD)-ROMドライブを接続する
  • LANケーブルを接続する
  • 電源を入れる
    • 「boot: 」と出たらキー入力する

ネットワークの確認

ネットワークに接続されていることを確認する。

# ping gentoo.org -c 3
PING gentoo.org (89.16.167.134) 56(84) bytes of data.
64 bytes from www.gentoo.org (89.16.167.134): icmp_seq=1 ttl=42 time=275 ms
64 bytes from www.gentoo.org (89.16.167.134): icmp_seq=2 ttl=42 time=275 ms
64 bytes from www.gentoo.org (89.16.167.134): icmp_seq=3 ttl=42 time=280 ms

--- gentoo.org ping statistics ---
3 packets transmitted. 3 received, 0% packet loss. time 2002ms
rtt min/avg/max/mdev = 275.072/276.910/280.478/2.523 ms

日付・時刻の修正

日付・時刻を確認し修正する。

# date
Sat May 30 02:20:51 UTC 2015
# /usr/init.d/ntp-client start
* Setting clock via the NTP client 'ntpdate' ...
# date
Sat May 30 02:21:17 UTC 2015

SWAPパーティションをONにする

# swwaon /dev/sda2
# free -h
...
Swap: 2.1G 0B 2.1G

基本的なファイルシステムのマウント

今回は/dev/sda1を/mnt/gentoo/に/dev/sdb1を/mnt/gentoo/usrにマウントすれば良い。

# mount /dev/sda1 /mnt/gentoo/
# mount /dev/sdb1 /mnt/gentoo/usr/

resolv.confのコピー

DNSサーバの位置が新しい環境でも見つけられるようにする。

# cp -L /etc/resolv.conf /mnt/gentoo/etc/

必要なファイルシステムのマウント

/proc/, /sys/, /dev/をそれぞれマウントする。

# mount -t proc proc /mnt/gentoo/proc
# mount --rbind /sys /mnt/gentoo/sys
# mount --make-rslave /mnt/gentoo/sys
# mount --rbind /dev /mnt/gentoo/dev
# mount --make-rslave /mnt/gentoo/dev

chroot

# chroot /mnt/gentoo /bin/bash
# source /etc/profile
# export PS1="(chroot) $PS1"

おめでとう

これでインストール作業を再開することができる。
