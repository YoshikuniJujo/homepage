安定結婚問題

(工事中 0%)

問題定義

現在組んでいる相手よりも「互いに」より好きであるペアが存在しないマッチングを 安定なマッチングと呼ぶ。 わかりやすく言うと「かけおち」が発生しない結婚の組み合わせということになる。 このようなマッチングを見つけようというのが安定結婚問題である。

Gale-Shapleyアルゴリズム

  • 独身の男性が存在する限り以下をくりかえす
    • まだプロポーズしていない女性のなかで最も好きな女性にプロポーズ
    • その女性に婚約相手がいなければ2人は婚約する
    • 婚約相手がいた場合
      • その婚約相手よりも新しい相手が好きならば2人は婚約
      • そうでなければ婚約はそのまま
  • すべての男性が結婚するまでくりかえす

Haskellによる実装

扱うデータ型

汎用性を考えてここでは男性と女性がそれぞれ互いに点数をつけることにする。 男性、女性、得点はどれも大小関係が決まる値なら何でも良いことにする。 男性、女性の値は本来なら同値関係のみで良いがソート可能な値としたほうが コードが単純化できる。 以下のような型とする。

  • 男性: Ord m => m
  • 女性: Ord w => w
  • 男性がつけた得点: Ord pm => pm
  • 女性がつけた得点: Ord pw => pw

入力データ

男性側、女性側のそれぞれの得点表が入力となる。

  • 男性側の得点表: [(m, [(w, pm)])]
  • 女性側の得点表: [(w, [(m, pw)])]

出力データ

それぞれのペアとその得点のリストが出力となる。

[((m, pm), (w, pw))]
