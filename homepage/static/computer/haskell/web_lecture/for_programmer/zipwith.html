関数zipWithの紹介

はじめに

「リストの再帰的定義」で使う関数zipWithの紹介をする。 定義について学ぶのはあとにする。 ここでは使いかたのみを扱う。

動作

関数mapの扱うリストをふたつにしたものだ。

map :: (a -> b) -> [a] -> [b]

zipWith :: (a -> b -> c) -> [a] -> [b] -> [c]

2引数関数とふたつのリストをとり、リストから1つずつ値をとりだして関数に与え結果を リストにまとめる。

% ghci
Prelude> zipWith (+) [3, 4, 5] [6, 7, 8, 9]
[9,11,13]

それぞれのリストから0番目、1番目、2番目、...と値をとりだして関数に与えていく。 短いほうのリストの要素がなくなったところで終了する。 長いほうのリストの残りの要素は無視される。

まとめ

関数zipWithの動作を見た。 関数mapのリストがふたつになったものだ。
