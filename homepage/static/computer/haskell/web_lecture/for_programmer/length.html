リストの長さ

動作

関数lengthはリストの長さを計算する。

% ghci
Prelude> length [1, 2, 3]
3
Prelude> length "hello"
5

考えかた

空リストの長さは0である。 リストの先頭に要素を追加すると長さは1増える。

コード

基底部

myLength [] = 0

再帰部

myLength (_ : xs) = 1 + myLength xs

リストの長さはなかの要素には関係ないのでワイルドカードでマッチしている。

型

型はどうなるだろうか。 「長さ」はなかの値の型を問わない構造に関する性質である。 よって多相関数とすることができるはずだ。

myLength :: [a] -> Int

返り値はIntegerでも良いが今回は関数lengthに合わせてIntとしておく。

まとめ

関数mySumとmyLengthを比較するとよく似ていることに気づく。 関数mySumは足し算の第1引数のxを1に置き換えると関数myLengthになる。 課題でそれを確認してみよう。

課題

    • 値をすべて1に変換したあと総和をとる
