演算子

演算子は関数である

Haskellでは演算子は本質的に関数と同じである。 表記上の違いがあるだけだ。 3 + 8という表現は関数(+)に3と8を与えたということ。 これは実際に確認できる。 

% ghci
Prelude> 3 + 8
11
Prelude> (+) 3 8
11

演算子は丸括弧で囲ってやることで関数として扱うことができる。

関数は演算子である

前に出てきた剰余を求める関数modがある。 これを今までは中置演算子として使ってきたが、これは普通の関数である。 関数はバッククォートで囲ってやることで演算子として使うことができる。 

Prelude> mod 15 4
3
Prelude> 15 `mod` 4
3

演算子の部分適用

演算子に関しては左側(第1引数)、右側(第2引数)のどちらについても部分適用が可能だ。 

Prelude> (3 +) 8
11
Prelude> (+ 8) 3
11

(3 +)は(+)の第1引数のみに値3を与えたものである。 よって与えられた引数を3に足す関数が返る。 (+ 8)は(+)の第2引数のみに値8を与えたものである。 これは与えられた引数に8を足す関数となる。

演算子化した関数への部分適用

演算子化した関数も同様の部分適用ができる。

Prelude> (`mod` 4) 15
3
Prelude> (15 `mod`) 4
3

演算子の定義

演算子の定義は関数定義と同じようにすれば良い。

(+++) a b = ...

中置記法を使うこともできる。

a +++ b = ...

演算子に使える記号

演算子にはアスキー記号の範囲内では以下のものが使える。

!#$%&*+./<=>?@\^|-~:

ただし':'ではじまる記号列は別の用途で使われるため、普通の演算子としては使えない。

関数の演算子的な定義

関数も演算子的な記法で定義することができる。 以下のような定義を見てみよう。 

op a b = ...

これを以下のように書くことができる。

a `op` b = ...
