演算子

演算子とは

演算子とは特殊な記法が可能な2引数関数と考えられる。 普通の関数は前置記法であるのに対して演算子は中置記法となる。

演算子の前後に置かれた値をそれぞれ第1、第2引数と考えることができる。

演算子は関数である

演算子は表記上の違い以外は関数と同じだ。 3 + 8は関数(+)に3と8を与えるのと同じだ。 確認してみよう。 

% ghci
Prelude> 3 + 8
11
Prelude> (+) 3 8
11

演算子は丸括弧で囲えば関数になる。

関数は演算子である

剰余を求める関数modがある。 中置演算子として使ってきたがこれは普通の関数だ。 

Prelude> mod 15 4
3
Prelude> 15 `mod` 4
3

関数はバッククォートで囲うと演算子になる。

演算子の部分適用

演算子では左側(第1引数)、右側(第2引数)のどちらについても部分適用できる。 

Prelude> (3 +) 8
11
Prelude> (+ 8) 3
11

(3 +)は(+)の第1引数のみに値3を与えたものだ。 与えられた引数を3に足す関数が返る。 (+ 8)は(+)の第2引数のみに値8を与えたものだ。 与えられた引数に8を足す関数となる。

演算子化した関数への部分適用

演算子化した関数にも同様の部分適用ができる。

Prelude> (`mod` 4) 15
3
Prelude> (15 `mod`) 4
3

演算子の定義

演算子の定義は関数定義と同じようにすれば良い。

(+++) a b = ...

中置記法を使っても良い。

a +++ b = ...

演算子に使える記号

演算子にはアスキー記号の範囲内では以下のものが使える。

!#$%&*+./<=>?@\^|-~:

ただし':'ではじまる記号列は別の用途で使われるため普通の演算子としては使えない。

コードの短縮

コードゴルフという競技がある。 与えられたお題でどれだけ短いコードが書けるかという競争だ。 短くするのに長い関数に短い別名をつけるという手法がある。 たとえば以下のようなコードがあったとする。

f x y=someLongFunction x+someLongFunction y

43文字である。 このコードは以下のようにすることで文字数を減らすことができる。

l=someLongFunction
f x y=l x+l y

これは別名の定義に必要な改行を入れて32文字だ。 11文字減らすことができた。 この変換によって減らすことのできる文字数は 関数の長さとそれが使われている回数から計算できる。

もとの関数の長さをlenとしその関数の使われている場所の数をtmsとする。 その関数を1文字の関数に置き換えることで減らせる文字数は(len - 1) * tmsだ。 1文字の別名をつけるために改行も含めて(len + 3)文字使う。 この変換によって減らせる文字数は以下のような式となる。

(len - 1) * tms - (len + 3)

演算子の名前を考える。 この演算は掛け算に似ている。 それに「減らす」イメージを追加して\*\とする。

golf.hs

len \*\ tms = (len - 1) * tms - (len + 3)

試してみよう。 上記の例ではsomeLongFunctionが16文字で2回使われていた。 この場合に11文字減らすことができる。

ghci golf.hs
*Main> 16 \*\ 2
11
*Main> 3 \*\ 3
0
*Main> 3 \*\ 2
-2

3文字の関数を3回使っている場合には変換によって文字数は変化せず、 3文字の関数を2回使っている場合には変換によってむしろ2文字増えてしまう。

関数の演算子的な定義

関数も演算子的な記法で定義できる。

op a b = ...

上記の定義を以下のように書ける。

a `op` b = ...

まとめ

演算子は記号列で表現され中置記法となる。 記法以外は2引数関数と同じだ。 演算子と関数は()と``で互いに変換できる。 演算子の定義にも中置記法が使える。
