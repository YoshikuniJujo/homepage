演算子

演算子とは

演算子は2引数関数だ。 ふつうの関数は前置記法だが演算子は中置記法だ。

演算子の前後の値が第1、第2引数となる。

演算子は関数だ

演算子と関数は表記だけが違う。3 + 8は関数(+)に3と8を与えるのと同じだ。

% ghci
Prelude> 3 + 8
11
Prelude> (+) 3 8
11

演算子を丸括弧ではさむと関数になる。

関数は演算子だ

関数modは剰余を計算する。ふつうの関数だ。

Prelude> mod 15 4
3
Prelude> 15 `mod` 4
3

関数はバッククォートではさむと演算子になる。

演算子の部分適用

演算子は左側(第1引数)でも右側(第2引数)でも部分適用できる。

Prelude> (3 +) 8
11
Prelude> (+ 8) 3
11

(3 +)は(+)の第1引数に値3を与えたものだ。引数を3に足す。 (+ 8)は(+)の第2引数に値8を与えたものだ。引数に8を足す。

演算子化した関数への部分適用

Prelude> (`mod` 4) 15
3
Prelude> (15 `mod`) 4
3

演算子化して部分適用する手法は第2引数について部分適用したいときに便利だ。

演算子の定義

演算子の定義は関数定義と同じようにする。

(+++) a b = ...

中置記法も使える。

a +++ b = ...

演算子に使える記号

アスキー記号の範囲で以下が使える。

!#$%&*+./<=>?@\^|-~:

:ではじまる記号列は別の用途で使われるためふつうの演算子としては使えない。

コードの短縮

コードゴルフという遊びがある。 与えられたお題でどれだけ短いコードが書けるかという競争だ。 短くするのに長い関数に短い別名をつけるという方法がある。

f x y=someLongFunction x+someLongFunction y

43文字だ。以下のようにして文字数を減らせる。

l=someLongFunction
f x y=l x+l y

定義に必要な改行をいれて32文字だ。11文字減らせた。 減らせる文字数は関数の長さと使われている回数から計算できる。

関数名の長さをlenとし使われている回数をtmsとする。 1文字の別名に置き換えて減らせる文字数は(len - 1) * tmsだ。 別名をつけるために改行も含めて(len + 3)文字使う。 よって、全体として減らせる文字数は以下のようになる。

(len - 1) * tms - (len + 3)

演算子の名前を考える。この演算は掛け算に似ている。 「減らす」イメージを加味して\*\とする。

golf.hs

len \*\ tms = (len - 1) * tms - (len + 3)

例では関数名someLongFunctionは16文字で使われている回数は2回だ。11文字減らせる。

ghci golf.hs
*Main> 16 \*\ 2
11
*Main> 3 \*\ 3
0
*Main> 3 \*\ 2
-2

3文字の関数を3回使っているとき文字数は変わらない。 3文字の関数を2回使っているときはむしろ2文字増える。

関数の演算子的な定義

関数は演算子的に定義できる。以下のように書き換えられる。

op a b = ...

a `op` b = ...

まとめ

演算子は記号列で表現され中置記法だ。 2引数関数との違いは記法だけだ。 演算子と関数は()と``で互いに変換できる。 演算子は定義にも中置記法が使える。
