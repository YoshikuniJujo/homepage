非カリー化

非カリー化とは

タプルとして引数を受けとる関数を、 ばらばらの引数を受けとる関数に変換することをカリー化と呼び、 関数curryが用意されていた。 逆に、ばらばらの引数を受けとる関数を、 タプルとして引数を受けとる関数に変換することを非カリー化と呼び、 関数uncurryが用意されている。

整数の組の足し算

2つで1組の値に対して、ばらばらの引数を取る関数を適用したいことがある。 たとえば整数の組に対して、その2つの値を足した値が欲しいような場合があるだろう。 そのようなとき、uncurryを使うとスマートである。 

% ghci
Prelude> uncurry (+) (3, 8)
11

関数uncurry

型

整数の組の足し算の例では、 Integer -> Integer -> Integerを (Integer, Integer) -> Integerに変換するので、 型は以下のようになる。 

(Integer -> Integer -> Integer) -> ((Integer, Integer) -> Integer)

より一般的には以下のようになる。

(a -> b -> c) -> ((a, b) -> c)

右側の丸括弧は省略できるので以下のようになる。

(a -> b -> c) -> (a, b) -> c

定義

uncurryの定義は以下のようになる。

uncurry :: (a -> b -> c) -> (a, b) -> c
uncurry f (x, y) = f x y

関数uncurryの第2引数であるタプル(x, y)の第1、第2要素をばらばらにして、 関数uncurryの第1引数である関数fに与えている。
