はじめに

値の「評価」のしくみをつくるためにまずは初期環境に適当な変数を定義し その変数の値を「評価」してみる。

シンボル

値構築子Symbol

data Value
| Symbol Symbol
= Int Integer

showValue (Symbol s) = s
showValue (Int i) = show i

トークン

data Token
= TkSym Symbol
| TkInt Integer
deriving Show

字句解析

シンボルに使える文字であることを確認する関数isSymChを定義する。

isSymCh :: Char -> Bool
isSymCh c = any ($ c) [isAlphaNum, (`elem` "+-*/<=>?")]

字句解析器を拡張する。

tokens (c : s)
| isDigit c = ...
| isSymCh c = let (t, r) = span isSymCh s in
(TkSym (c : t) :) `mapply` tokens r
| isSpace c = ...
tokens "" = Just []
tokens _ = Nothing

構文解析

シンボルをとりだす。

parse1 (TkSym s : ts) = Just (Symbol s, ts)

の行を追加する。

評価

テスト用変数

Primitive.hs

テスト用の変数を初期環境に定義する。

env0 = fromList [
("hoge", Int 12345)
]

変数値の参照

Environment.hs

変数の値を環境からとりだす関数referを定義する。

refer :: Symbol -> Env -> Maybe Value
refer = M.lookup

関数referはData.Mapモジュールの関数lookupがそのまま使える。 関数referを公開する。

module Environment (Env, M.fromList, refer, Value(..), showValue, Symbol) where

関数eval

Eval.hs

モジュールEvalを作り評価関数を作る。

{-# LANGUAGE TupleSections #-}

module Eval (evaluate) where

import Environment
import Maybe

eval :: Value -> Env -> Maybe (Value, Env)
eval (Symbol s) e = (, e) `mapply` refer s e
eval i@(Int _) e = Just (i, e)

関数evaluate

Value型のリストの各要素を次々に評価していく関数evaluateを作る。

evaluate :: [Value] -> Env -> Maybe ([Value], Env)
evaluate [] e = Just ([], e)
evaluate (v : vs) e = case eval v e of
Just (v', e') -> (\(vs', e'') -> (v' : vs', e'')) `mapply` evaluate vs e'
_ -> Nothing

モジュールMain

nsc.hs

まずはモジュールEvalを導入する。

import Eval

関数schemeにevaluateの段階を追加する。

scheme src e = (`evaluate` e) `mbind` (parse `mbind` tokens src)
