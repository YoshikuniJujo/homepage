相互再帰

再帰が「自身を呼ぶ」のに対して相互再帰は「互いに呼びあう」。 その相互再帰のシンプルで美しい例を示す。

パーサをリストのパーサに

ひとつの要素をパースするパーサを複数の要素をパースしてリストとして返すパーサにする。 ここでポイントは0個以上のリストをパースするパーサと 1個以上のリストをパースするパーサを同時に作ることだ。

list, list1 :: Parse a -> Parse [a]
list p = succeed [] `alt` list1 p
list1 p = (p >*> list p) `build` uncurry (:)

`build` uncurry (:)の部分は(x, xs)のようなタプルをx : xsのようにして リストにしているだけだ。 一度これを消して考える。

list p = succeed [] `alt` list1 p
list1 p = p >*> list p

として考えてみよう。 ある要素の0個以上のリストは「0個のリストまたは1個以上のリスト」だ。 ある要素の1個以上のリストは「ある要素に0個以上のリストを続けたもの」だ。 筋が通っている。

文字列を与えたときのことを考える。 そのときにパーサpがパースできなくなったとき パーサlist1 pによる候補は空となる。 これを基底と考えることができる。 そしてパーサpが必ず文字をひとつ以上消費するのであれば、 p >*> list pの後者のlist pの扱う文字列は必ず減少する。 よって基底に近づくことがわかる。

再帰関数は「筋が通っ」ていて、かつ、再帰のたびに「基底に近づく」ならば 問題なく定義されていることになる。 よってこれらの関数はパーサpが必ず1文字以上消費するならば問題なく動作する。

動作

list (check isDigit) "123"
[("","123"),("1","23"),("12","3"),("123","")]

[("1","23"),("12","3"),("123","")]

まとめ

単一要素のパーサをリストのパーサにする関数は相互再帰の美しい例だ。
