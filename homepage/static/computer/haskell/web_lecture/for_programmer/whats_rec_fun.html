再帰関数とは

再帰という概念

あるモノを表すうえでそのモノ自体が参照されるような場合に、 そのような表現を再帰的であると言う。

再帰は関数型言語の魂である。 今まで見てきたリストも再帰によって作成される。 「くりかえし」は再帰によって実現できる。 樹構造であれ何であれ再帰によって表現可能だ。

「何でもできる」。 非常に強力な概念である。 しかし、強力であるということはときに「わかりにくさ」につながることがある。 前に学んだ「列挙 - 蓄積」という枠組みでできることは、 その枠組みでやったほうが良い。

再帰という考えかたはリストをはじめ、様々な概念の下に存在する。 再帰を理解すれば、様々に異なる概念の根っ子を理解することができる。 再帰を理解することは必須だ。 しかし、実際にコードを書く際には、 道具箱のなかから、リスト等のより力の弱い概念を使うことを試み、 それらが使えなかったときのみ、 生の再帰を使用することを考えるほうが良い。 そのほうがわかりやすいコードが書けることが多いだろう。

再帰の例

たとえば自然数とは何かと聞かれたとき、以下のように答えることができる。 「1は自然数であり、自然数に1を加えた数も自然数である」。 これは再帰的な定義の例である。

再帰関数

その定義のなかで自分自身を参照している関数を再帰関数と呼ぶ。 Haskellでは自分で定義した関数であれ、誰かが定義した関数であれ、 多くの関数が再帰関数である。 再帰関数について学んでいくことにする。
