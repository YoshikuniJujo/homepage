ライプニッツの公式(円周率を求める)

はじめに

ちょっとしたジレンマがある。 初学者にわかりやすい例を使いたい。 しかし、初学者にわかりやすい例だと生の再帰を使うメリットがない。 リストを使ったほうがきれいに書ける。 リストを使ったほうが良い例にあえて生の再帰を使ってみよう。 次にリストでは解決できない例を見る。

ライプニッツの公式とは

円周率を4で割った値は以下のようになる。 証明はしない。

1 - 1 / 3 + 1 / 5 - 1 / 7 + 1 / 9 - ...

符号を反転させながら分母を[1, 3, 5, 7, 9 ...]と増やす。 単純な級数だ。 収束が遅いため実用的ではないが例題としては使いやすい。

再帰関数を使う

leibnizRec.hs

円周率の4分の1がわかれば

「...が定義されていれば」という仮定に慣れることが再帰関数を理解するコツだ。 再帰関数でない関数で練習しよう。 「円周率の4分の1の近似値を求める関数estPi4が定義されていれば」 円周率の近似値を求める関数estPiは以下のように定義できる。

estPi :: Double -> Double
estPi n = 4 * estPi4 n

級数の何個目まで計算するかを示す引数を取る。 「何個」なので整数値とするべきだが計算の便宜を考えて実数値Doubleとした。 定義から仮引数nを消せる。

estPi = (4 *) . estPi4

仮引数を使わない書きかたをポイントフリースタイルと呼ぶ。 ポイントフリースタイルを使うと直接的な表現になる。 「estPiはestPi4してから4倍する関数」。

関数estPi4

関数estPi4に0, 1, 2 ...と引数を与えた結果を見る。

estPi4 0 = 1
estPi4 1 = 1 - 1 / 3
estPi4 2 = 1 - 1 / 3 + 1 / 5
estPi4 3 = 1 - 1 / 3 + 1 / 5 - 1 / 7
estPi4 4 = 1 - 1 / 3 + 1 / 5 - 1 / 7 + 1 / 9
...

隣接する値同士を比較する。

estPi4 0 = 1
estPi4 1 = estPi4 0 - 1 / 3
estPi4 2 = estPi4 1 + 1 / 5
estPi4 3 = estPi4 2 - 1 / 7
estPi4 4 = estPi4 3 + 1 / 9
...

引数が奇数のときには引き算を、偶数のときには足し算を行っている。 引いたり足したりする数は引数をnとすると2 * n + 1だ。

estPi4 0 = 1
estPi4 1 = estPi4 0 + (- 1) ** 1 / (2 * 1 + 1)
estPi4 2 = estPi4 1 + (- 1) ** 2 / (2 * 2 + 1)
estPi4 3 = estPi4 2 + (- 1) ** 3 / (2 * 3 + 1)
estPi4 4 = estPi4 3 + (- 1) ** 4 / (2 * 4 + 1)
...
estPi4 n = estPi4 (n - 1) + (- 1) ** n / (2 * n + 1)

(**)は第2引数が実数のべき乗演算子だ。 第2引数が整数のべき乗演算子は(^)だ。 これらは意味が異なるので別々の演算子となっている。 estPi4 0以外はnを使った表現にまとめられる。 estPi4 0とestPi4 nだけを抜き出すと実際のHaskellの関数定義となる。

estPi4 0 = 1
estPi4 n = estPi4 (n - 1) + (- 1) ** n / (2 * n + 1)

リストを使う

leibnizList.hs

学習のために上では生の再帰を使った。 この例ではリストを利用したほうが「きれい」だ。

0以上の整数のリストの要素すべてを以下の関数で置換する。

\k -> (- 1) ** k / (2 * k + 1)

結果としてできたリストからn個取り出し総和を求める。

estPi n = sum . take n $ map (\k -> (- 1) ** k / (2 * k + 1)) [0 ..]

ポイントフリースタイルにする。

estPi = sum . (`take` map (\k -> (- 1) ** k / (2 * k + 1)) [0 ..])

生の再帰を使うバージョンにくらべてコーディングの文字数自体は増えているが、 こちらのほうが「きれい」だ。

まとめ

再帰関数の定義のしかたを学んだ。 基底(今回は0)と漸化式を定義する。 自然数の範囲でのすべての値が求まる。 単純な「くりかえし」なのでリストを使ったほうが簡単だ。
