関数リテラル

リテラルとは

'c', "hello", 123のようにコードのなかに値を「直接」書きこむ表現がリテラルだ。 変数や関数適用はリテラルではない。

% ghci
Prelude> 'c'
'c'
Prelude> "hello"
"hello"
Prelude> 123
123

関数リテラルとは

ここまでで学んだ定義では関数は変数に束縛する必要があった。 関数リテラルなら関数に名前をつけずに使える。

変数とリテラル

x = 3と定義されているとき変数xを3で置き換えられる。 では、fun x = x ^ xのように定義されているとき 変数funを何で置き換えられるだろうか。 funを置き換えられるような表現、 関数自体をそのまま書いたもの、が関数リテラルだ。

関数リテラルの例

% ghci
Prelude> let fun x = x ^ x
Prelude> fun 3
27

funの部分を関数リテラルで置き換える。 関数リテラルは\[仮引数] -> [表現]という形になる。

Prelude> (\x -> x ^ x) 3
27

関数定義は構文糖だ

今まで見てきた関数定義は2つのことを行う構文糖だ。

fun x = x ^ x

以下のように書き直せる。

fun = \x -> x ^ x

関数定義は関数の作成とその関数での変数の束縛という2つのことを行う。

xxcryptの例

xxcrypt n ed mc = mc ^ ed `mod` n

関数リテラルで書く。

xxcrypt = \n -> (\ed -> (\mc -> mc ^ ed `mod` n))

数値を返す関数を返す関数を返す関数だ。 結合性を考慮して丸括弧を省略する。

xxcrypt = \n -> \ed -> \mc -> mc ^ ed `mod` n

複数引数の略記法

xxcrypt = \n ed mc -> mc ^ ed `mod` n

まとめ

\[仮引数1] [仮引数2] ... -> [表現]

課題

  1. 金額を引数とし消費税込みの価格を返す関数taxを作成せよ
    • 税率は8%とする
    • 整数の除算には演算子`div`が使える
    • 20 `div` 3は6となる
  2. 関数taxを関数リテラルで変数を束縛する形に書き換えよ
  3. 時間、分、秒を引数としてとり秒数に変換する関数secを作成せよ
  4. 関数secを関数リテラルで変数束縛する形に書き換えよ
