1から学ぶHaskell: はじめに

Haskellを学ぼう。 「Haskellを学ぶ」ということは「新たな言語を学ぶ」ということ以上の価値がある。 手続き型言語しか知らなかった人が オブジェクト志向を学んだときに感じるのと同等な驚きがある。 海の向こうにも国があると初めて知ったときの驚きだ。

「関数型言語」と呼ばれる言語は多々あるがHaskellは一味違う。 Haskellには定冠詞をつけて「ザ・関数型言語」と呼べるだろう。

純粋性と実用性とはしばしば相反する。 しかし、ときどき、それが同じ向きになることがある。 たとえばHaskellの採用した「参照透過性」という性質がある。 「関数に同じ引数を渡したら常に同じ値が返る」ということだ。 これは他の言語を使ってきた人達には偏狭な純粋主義のように見える。 しかし、この性質によって、コードは簡潔になり機械的な最適化も可能になる。

「静的型付け」という性質もある。 「型」はプログラマにとって有用な道しるべとなりバグを減らす効果があるだけでなく、 実効効率を改善するという効果もある。

「純粋性」を追求したら「実用性」も高くなった。 そこにHaskellの美しさがある。 美しいデザインを持つものはしばしば機能的にも優れている。 他の言語が場当たり的に行っている改善を言語の中核としてはじめから持っている。 それがHaskellなのだ。
