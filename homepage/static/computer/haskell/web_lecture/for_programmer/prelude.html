1から学ぶHaskell: はじめに

Haskellを学ぼう。 「Haskellを学ぶ」ということは単に「新たな言語を学ぶ」ということ以上の価値がある。 そこには、手続き型言語しか知らなかった人が、 オブジェクト志向について学んだときに感じるような驚きがある。 あるいは、海の向こうにも国があると初めて知ったときのような驚きと言ってもいい。

「関数型言語」と呼ばれる言語は多々あるけれど、Haskellはまた一味違う。 Haskellには定冠詞をつけて「ザ・関数型言語」と呼ぶべきだ。

純粋性と実用性とはしばしば相反する。 しかし、ときどき、それが同じ向きになることがある。 たとえばHaskellの採用した「参照透過性」という性質がある。 これは「関数に同じ引数を渡したら常に同じ値が返る」ということだ。 これは他の言語を使ってきた人達には偏狭な純粋主義のように見える。 しかし、これがあることで、コードは簡潔になり 機械的な最適化も可能になる。

他の例としては「静的型付け」という性質もある。 「型」はプログラマにとって有用な道しるべとなり、 バグを減らす効果があるだけでなく、 実効効率を改善するという効果もある。

「純粋性」を追求したら「実用性」も高まった。 そこにHaskellの美しさがある。 美しいデザインを持つものは、しばしば機能的にも優れている。 他の言語が場当たり的に行っている改善を、 むしろ言語の中核の部分においてはじめから持っている。 それがHaskellなのだ。
