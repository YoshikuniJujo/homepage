関数takeToの関数unfoldrによる定義

takeTo_unfoldr.hs

はじめに

抽象化のしかたはひとつではない。 それが面白いところだ。 関数takeToは以下のような型だ。

takeTo :: (a -> Bool) -> [a] -> [a]

「リストを引数にとる関数」でもあり「リストを返す関数」でもある。 前者としてfoldrを使って定義したが後者としてunfoldrを使って定義することもできる。

生の再帰

takeTo p [] = []
takeTo p (x : xs)
| p x = [x]
| otherwise = x : takeTo p xs

xが条件を満たすならそれ以降の要素は不要なのでxだけから成るリストを返す。 満たさないならば再帰的にtakeTo pを残りのリストに適用した結果にxを追加する。

関数foldrによる定義

takeTo p = foldr (\x -> (x :) . if p x then const [] else id) []

xが条件を満たしたときは続く値を問わず空リスト(const [])に、 満たさないならば続く値に再帰的に関数を適用した結果(id)に、 xを追加した値を返す。 引数であるリストの先頭に次々と値を追加していき値を変化させていくイメージだ。

関数unfoldrによる定義

モジュールData.Listが必要だ。

import Data.List

takeTo = unfoldr $ \s -> case s of
[] -> Nothing
x : xs | p x -> Just (x, [])
| otherwise -> Just (x, xs)

リストの先頭を結果として返しながら要素を先頭から削っていくイメージだ。 xが条件pを満たしたときは次にわたすリストは空リストとする。

まとめ

関数takeToを「リストをとる関数」として、あるいは「リストを返す関数」として、 の2通りで抽象化できることを学んだ。 暗記するのではなくたくさんの例を見ていくことで体で覚えることが大切だ。
