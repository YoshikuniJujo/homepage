Haskellを学ぶ動機

Haskellを学ぶ動機を挙げていく。

将来性

関数型以前、あるいはHaskell以前の言語は、 機械側の都合によって様々な概念がねじまげられていた。 そして、そのねじまげられた概念に合わせるようにして、プログラムは組まれてきた。 ある程度の規模まではその歪みは顕在化することがない。 しかし、機械の能力が拡大するにつれてコードも肥大化する。 肥大化したときに、概念の歪みが牙をむく。 保守不可能なコードが次々と生まれている。

Haskellの考えかたは、むしろ自然だ。 既存の言語を知らない頭で考えれば、アルゴリズムを最も自然に記述できる言語である。 原理的な部分に近い素直な記述であることを反映して、 コードが肥大化したときも保守可能なままに保つことがたやすい。

また、現在必須の技術として、並行・並列処理というものがあるが、 むやみな状態変化を許さないHaskellでは意味を変化させずに 並行化・並列化させることが可能だ。 その意味でも今後有望な言語と言えるだろう。

学問的

Haskellは根本的なところが学問的である。 様々な機能が論文などによって、その正しさや問題点を検証されている。 進歩し続ける数学的研究の成果の数々を利用することで、 機能の選択やその記法等における選択において正しい道を歩んでいるように思われる。

実用性

学問的であるということと実用性というものが相反するものととらえられているが、 実際はそうではない。 Haskellにおいて学問と実用とは美しく融合している。

何かの機能が簡潔に書けるとする。 他の言語では場当たり的な略記法だったりするのだが、 Haskellでは「簡潔に書ける理由」に学問的な正当性がある。

楽しい

Haskellを学ぶ一番の理由は、もちろん、楽しさである。 低レベルな機械の都合に左右されることなく、 自分の書きたいロジックをそのまま書くことができる。

また、さまざまな研究の成果が取り込まれているので、 そのような学問的努力の産物を味わうことができる。 しかも、ただ読んで理解するというだけではなく、 実際に手を動かして確かめながら理解していくことができる。
