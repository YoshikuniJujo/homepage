リストの要素の総和

はじめに

再帰関数を学び、リストのパターンマッチを学んだので、 リスト引数として取る再帰関数を定義することができるようになった。 まずは簡単な例から見ていく。 リストの総和を求める関数sumを自分で定義してみよう。

関数sum

総和を求める関数sumが用意されている。 前にも使ったので覚えているだろう。

% ghci
Prelude> sum [3, 8, 2]
13

関数mySum

型

関数mySumはInteger型のリストからその総和を求めるものとする。 よって型は以下のようになる。

mySum :: [Integer] -> Integer

定義

リストとは「リストの先頭に要素を追加したもの」または「空リスト」なので リストを引数に取る関数はその2つでパターンマッチすれば良い。

空リスト

まずは空リストから。

mySum [] = ...

空のリストの総和は何だろうか。 [3, 8, 2]の総和は13、[8, 2]の総和は10、[2]の総和は2、それなら[]の総和は何だろう。 リストからひとつ数を消すとその消した数のぶんだけ総和は減少する。 よって[]の総和は[2]の総和から2を引いた値ならば良い。 つまり0だ。 「空リストの総和は0である」というよりもむしろ 「空リストの総和を0とすると都合が良い」ということだ。

mySum [] = 0

空でないリスト

空でないリストについて考えてみよう。 [3, 8, 2]の総和は[8, 2]の総和に3を足したもの、 [8, 2]の総和は[2]の総和に8を足したもの、 [2]の総和は[]の総和に2を足したもの。 以下のような関係が成り立つ。

mySum [3, 8, 2] == 3 + mySum [8, 2]
mySum [8, 2] == 8 + mySum [2]
mySum [2] == 2 + mySum []
mySum [] == 0

これを逆にたどればmySum [3, 8, 2]は13となる。 一般的に書くと以下のようになる。

mySum [x1, x2, ..., x3] == x1 + mySum [x2, ..., x3]

先頭の要素と残りのリストは(x : xs)のようなパターンマッチで取り出せる。

mySum (x : xs) = x + mySum xs

出来上がり

mySum.hs

mySum :: [Integer] -> Integer
mySum [] = 0
mySum (x : xs) = x + mySum xs

まとめ

リストの総和を求める再帰関数を作成した。 「空リストの総和は0である」と 「リストの総和は先頭の数と残りのリストの総和の和である」とを定義する。 リストを引数に取る再帰関数の多くがこの形で定義できる。 再帰関数を定義するときは以下のように考える。

  • 基底ケースは何か
  • 定義されている関数がすでに存在するものと考える
  • その関数をその関数自体でどう表現できるか
  • 引数が基底に近づいていくようにする

リストでは多くは空リストが基底となり、 先頭の要素を削除することで「基底に近づく」。
