多相関数とは

複数の型に適用可能

多相関数とは複数の型に適用可能であるか、複数の型のうちどれかの型を返すか、 またはその両方の性質を持つ関数だ。

2種類の多相関数

多相性にはパラメトリック多相とアドホック多相とがある。 アドホック多相は型クラスのところで説明する。 ここで扱うのはパラメトリック多相である。

パラメトリック多相

パラメトリック多相は型や関数の構造として値の中身が問題にならないために、 どの型でも取ることができるという意味での多相性だ。 Maybe型から値をとりだすfromMaybeという関数がある。 第1引数にデフォルト値を第2引数がMaybe値を与える。

% ghci
Prelude> :m Data.Maybe
Prelude Data.Maybe> fromMaybe 0 (Just 123)
123
Prelude Data.Maybe> fromMaybe 0 Nothing
0

Just値ならば値をとりだし、Nothingならデフォルトの値を返す。 この操作はMaybe型が包む型が何であろうとも関係ない。 関数の型は以下のようになる。

fromMaybe :: a -> Maybe a -> a

上記の'a'を型変数と呼ぶ。 型変数は小文字からはじまる識別子だ。 どの型でもこの部分に入れることができる。 同じ型変数には同じ型を入れる必要がある。 以下はfromMaybeの型として正しい。

Integer -> Maybe Integer -> Integer
Char -> Maybe Char -> Char
Bool -> Maybe Bool -> Bool

以下はfromMaybeの型として正しくない。

Integer -> Maybe Bool -> Char

制御構造として

多相関数は扱っている値自体ではなく構造全体に作用する関数だ。 多相関数によって実現される機能の多くはHaskell以前の言語では 制御構造として特殊な構文となっている。 Haskellでは他の言語で特殊な構文となっている制御構造を単なる関数として定義できる。

まとめ

Haskellでは値自体に影響しない関数を多相関数として定義できる。 値自体ではなく構造を扱う関数だ。 それらの多くは高階関数でもある。 広く使うことのできる制御構造を特殊な構文ではなく関数として定義できる。 多相関数の例をわかりやすいものから見ていこう。
