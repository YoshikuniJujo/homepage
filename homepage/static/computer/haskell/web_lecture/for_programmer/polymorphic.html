多相関数とは

複数の型に適用可能

関数が

  • 複数の型の値に適用可能である
  • 複数の型のうちどれかの型の値を返す

のどちらかまたは両方であるときその関数は多相関数である。

2種類の多相関数

多相性にはパラメトリック多相とアドホック多相とがある。 アドホック多相は型クラスで説明する。 ここではパラメトリック多相を扱う。

パラメトリック多相

パラメトリック多相は型や関数の構造として値のなかみが問題にならないために、 どの型の値であるかを問わないという意味での多相性だ。 Maybe型から値をとりだす関数fromMaybeがある。 第1引数にデフォルト値を第2引数にMaybe値を与える。

% ghci
Prelude> :m Data.Maybe
Prelude Data.Maybe> fromMaybe 0 (Just 123)
123
Prelude Data.Maybe> fromMaybe 0 Nothing
0

Just値からは値をとりだす。 Nothingならデフォルトの値を返す。 この操作はMaybeが包む値の型には関係ない。

fromMaybe :: a -> Maybe a -> a

'a'は型変数だ。 型変数には小文字からはじまる識別子が使われる。 ここにどの型をいれても良いということを示す。 同じ型変数には同じ型がはいる。

Integer -> Maybe Integer -> Integer
Char -> Maybe Char -> Char
Bool -> Maybe Bool -> Bool

同じ型変数に異なる型はいれられない。

Integer -> Maybe Bool -> Char (これはだめ)

制御構造として

多相関数は値自体ではなく構造をあつかう関数だ。 多相関数が実現する機能の多くはHaskell以前の言語では制御構造として特殊な構文となる。 Haskellでは他の言語で特殊な構文となる制御構造を単なる関数として定義できる。

まとめ

Haskellでは値自体の影響をうけない関数を多相関数として定義できる。 値自体ではなく構造を扱う。 多くは高階関数だ。 広く使うことのできる制御構造を特殊な構文ではなく関数として定義できる。 多相関数の例をわかりやすいものから見ていく。
