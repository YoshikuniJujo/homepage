型変換

はじめに

型変換を関数側で行うという例を示す。 「タプルを扱う関数」と「2引数関数」との相互変換について学ぶ前の準備として、 引数を変換する代わりに関数のほうを変換するという例を学ぶ。

値の変換

実数を半分にする関数を考える。

convert.hs

half :: Double -> Double
half = (/ 2)

また整数値7を定義する。

seven :: Integer
seven = 7

sevenにhalfを適用すると型エラーが生じる。

% ghci convert.hs
*Main> half seven

<interactive>:X:Y:
Couldn't match expected type `Double' with actual type `Integer'
In the first argument of `half', namely `seven'
In the expression: half seven
In an equation for `it' : it = half seven

「halfの第1引数であるsevenで 期待していたDoubleと実際のIntegerのあいだで型の不一致が起きた」。 型を合わせなければならない。 整数から実数に型を変換するのに関数fromIntegralが使える。 

*Main> half $ fromIntegral seven
3.5

これでIntegerからDoubleへの型変換ができた。 実際にはfromIntegralは整数を取り実数を返す単なる関数である。

関数を変換する関数

値ではなく関数のほうを変換してやれば値の型変換の必要はない。 関数の型をDouble -> DoubleからInt -> Doubleに変換する。 変換関数の型は以下のようになる。

(Double -> Double) -> (Integer -> Double)

'->'は右結合なので右側の丸括弧は省略できる。

(Double -> Double) -> Integer -> Double

第2引数のIntegerをDoubleに変換して第1引数である関数に与える関数となるはずだ。 以下をconvert.hsに追加しよう。

convert :: (Double -> Double) -> Integer -> Double
convert f n = f $ fromIntegral n

*Main> :reload
*Main> convert half seven
3.5

変換関数の本質

f $ fromIntegral nはnにfromIntegralを適用した結果にfを適用する。 よって(f . fromIntegral) nとなる。

convert f n = (f . fromIntegral) n

convert fをnに適用したものが(f . fromIntegral)をnに適用したものだ。 つまりconvert fはf . fromIntegralと同じだ。

convert f = f . fromIntegral

演算子の部分適用で仮引数fも省略できる。

convert = (. fromIntegral)

convert halfはhalf . fromIntegralだ。 関数halfの引数をDoubleからIntegerに変換するのにIntegerからDoubleへの変換関数を使う。

half :: Double -> Double
half . fromIntegral :: Integer -> Double

変換アダプタ

「値を変換するか関数を変換するか」は「変換アダプタをどちらにつけるか」だ。 

*Main> half (fromIntegral seven)
3.5
*Main> (half . fromIntegral) seven
3.5

まとめ

値を変換するかわりに関数を変換するという考えかたに注目しよう。 次に学ぶ関数curryや関数uncurryは「タプルにしたりばらばらにしたりする」変換を 値ではなく関数に対して行う。

課題

  2. 上記の変換を関数側に行うようにせよ
