型変換

はじめに

型変換を関数側で行うという例だ。 「タプルを扱う関数」と「2引数関数」との相互変換について学ぶ前の準備だ。 引数を変換するかわりに関数のほうを変換するという例を学ぶ。

値の変換

実数を半分にする関数を考える。

convert.hs

half :: Double -> Double
half = (/ 2)

整数値7で変数sevenを束縛する。

seven :: Integer
seven = 7

sevenにhalfを適用すると型エラーが生じる。

% ghci convert.hs
*Main> half seven

<interactive>:X:Y:
Couldn't match expected type `Double' with actual type `Integer'
In the first argument of `half', namely `seven'
In the expression: half seven
In an equation for `it' : it = half seven

「halfの第1引数のsevenに期待されるDoubleと実際のIntegerとのあいだで 型の不一致が起きた」。 関数fromIntegralで整数から実数への型変換ができる。

*Main> half $ fromIntegral seven
3.5

IntegerからDoubleへの型変換ができた。

関数を変換する関数

関数のほうを変換することを考える。。 関数の型をDouble -> DoubleからInt -> Doubleに変換する。 変換関数の型を示す。

(Double -> Double) -> (Integer -> Double)

->は右結合なので右側の丸括弧は省略できる。

(Double -> Double) -> Integer -> Double

第2引数のIntegerをDoubleに変換して第1引数である関数に与える関数となる。

convert :: (Double -> Double) -> Integer -> Double
convert f n = f $ fromIntegral n

*Main> :reload
*Main> convert half seven
3.5

変換関数の本質

f $ fromIntegral nはnにfromIntegralを適用した結果にfを適用する。 よって(f . fromIntegral) nだ。

convert f n = (f . fromIntegral) n

convert fをnに適用したものが(f . fromIntegral)をnに適用したものだ。 つまりconvert fはf . fromIntegralだ。

convert f = f . fromIntegral

演算子の部分適用で仮引数fも消せる。

convert = (. fromIntegral)

convert halfがhalf . fromIntegralであることがわかる。 関数halfの引数のDoubleからIntegerへの変換には 値を逆にIntegerからDoubleに変換する関数を使う。

half :: Double -> Double
half . fromIntegral :: Integer -> Double

変換アダプタ

「値を変換するか関数を変換するか」は「変換アダプタをどちらにつけるか」だ。

*Main> half (fromIntegral seven)
3.5
*Main> (half . fromIntegral) seven
3.5

まとめ

値を変換するかわりに関数を変換するという考えかたに注目する。 同じように関数curryや関数uncurryは「タプルにしたりばらばらにしたりする」変換を 値ではなく関数に対して行う。

課題

  1. 対話環境で文字'c'を文字コードに変換したものが偶数かどうかをチェックせよ
    • 文字コードへの変換には関数ordを使う
    • 偶数であることの確認には関数evenを使う
  2. 文字コードへの変換を関数even側でするようにせよ
