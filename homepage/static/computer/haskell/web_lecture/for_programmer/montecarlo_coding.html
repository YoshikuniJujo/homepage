モンテカルロ法のコーディング

montecarlo.hs

正方形内のランダムな点

正方形内のランダムな点を生成する式。

uncurry zip . (randomRs (-1, 1) *** randomRs (-1, 1)) . split $ mkStdGen 8

乱数の種を決めるための整数を引数とし いろいろな系列のランダムな点の列を返す関数とする。

points :: Int -> [(Double, Double)]
points = uncurry zip . (randomRs (-1, 1) *** randomRs (-1, 1)) . split . mkStdGen

円内の点であることのチェック

円内の点ということは原点からの距離が1以下ということ。

inCircle :: (Double, Double) -> Bool
inCircle (x, y) = x ^ 2 + y ^ 2 <= 1

円内の点のみを取り出す

全体の点の数を指定して円内の点のみを取り出す関数を作成する。

inCirclePoints :: Int -> Int -> [(Double, Double)]
inCirclePoints g n = filter inCircle . take n $ points g

円周率の推測値を出す

円内の点の数を全体の点の数で割ったものに正方形の面積である4をかける。

[正方形の面積] * [円内の点の数] / [正方形内の点の数]

ランダムな点の系列と点の数を指定し円周率の推測を計算する。

guessPi :: Int -> Int -> Double
guessPi g n = 4 * fromIntegral (length $ inCirclePoints g n) / fromIntegral n

試してみる

% ghci montecarlo.hs
*Main> guessPi 8 100

*Main> guessPi 8 1000

*Main> guessPi 9 1000000

グラフ

横軸に点の数、縦軸に円周率の推測値を取ったグラフ。 収束が遅いので横軸は対数。 gの値により異なる系列のランダム列を使っている。

πに収束しているように見える。

まとめ

モンテカルロ法で円周率を求める例題を見た。 ランダムな点を列挙した無限リストを作成。 円内にあるものを数え全体の点の数で割った値から円周率の近似値を求める。 関数型言語の魂である「再帰関数」についてまだ学んでいないが 「リストを使うだけでここまでできる」。
