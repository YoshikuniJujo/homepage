関数foldrによる定義について

はじめに

とくに関数foldrを使った定義には難しいものがいくつかある。 すぐにすべて理解する必要はない。 具体的な値で展開してみることをお勧めする。

ここから先はややこしいところがある。作成の途中なので余計だ。 進めるところまで進み、難しすぎるようだったら次回に回す。 整理しておくので。

ここに難しそうな項目の予備的な練習を載せよう

  • take, drop, splitAtの練習として foldrでたたみこんだ結果が1引数関数になるような例を考えることにする。
    1. [x1, x2, x3 ...]を[x1, x2, x2, x3, x3, x3, ...]にする関数とか?
    2. 上だとreplicateとか使ってややこしいので[x1 * 1, x2 * 2, x3 * 3, ...] としよう
  • dropWhile, spanの練習として foldrでたたみこんだ結果がBool値をとる関数になるような例を考える。 そのBool値はどこかの時点での条件の変化を示す。
  • zip, zipWithの練習として foldrでたたみこんだ結果がリストをとる関数になるような例を考える。

やはり

難しいものは「番外編」に移そう。 絶対に必要なものでもないし。
