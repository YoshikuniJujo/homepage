リストのパターンマッチ

リストは再帰的なデータ構造

リストの先頭に要素を追加

演算子(:)でリストの先頭に要素を追加できる。

% ghci
Prelude> 8 : [3, 5, 7]
[8, 3, 5, 7]
Prelude> 'H' : "ello"
"Hello"

空リスト

要素数0のリストは[]だ。

Prelude> []
[]

リストの構造

リストは先頭への追加演算子と空リストで作れる。

Prelude> 8 : 3 : 5 : 7 : []
[8, 3, 5, 7]

内部的にも8 : 3 : 5 : 7: []という形で保存されている。 [8, 3, 5, 7]という表記は対話環境で読みやすい形に直しただけだ。 リストは本質的にx : y : z : ... : []という構造のデータだ。

構文糖

[x, y, ... ,z]のような形では構文糖だ。 x : y : ... : z : []として解釈される。 演算子(:)は右結合だ。 丸括弧を省略しないとx : (y : (... : (z : []) ...))となる。

再帰的

「リストとはリストの先頭に要素を加えたもの、または空リストだ」。 再帰的な定義だ。疑似コードを示す。

[a] = (a : [a]) or []

Maybe a型の値がJust aまたはNothingなのと同じように[a]型の値はa : [a]または[]だ。

リストへのパターンマッチ

リストは次の2つのどちらかだ。

  • 要素 : リスト
  • 空リスト

2つのパターンでパターンマッチする。

  • x : xs
  • []

対象となるリストが空リストでなければx : xsにマッチし リストの先頭の要素がxを束縛しリストの残りの部分がxsを束縛する。 「先頭の要素」と「残りの要素から成るリスト」を取り出すことになる。 空リストは[]にマッチする。

構文糖

パターンマッチにも構文糖が使える。以下のようにパターンマッチできる。

f [x] = ...
g [x, y, z] = ...

それぞれ以下のように解釈される。

f (x : []) = ...
g (x : y : z : []) = ...

関数head

関数headはリストの先頭を取り出す。

Prelude> head [3, 4, 5]
3
Prelude> head "Hello"
'H'

関数myHeadを定義する。

myHead :: [a] -> a
myHead (x : _) = x

関数tail

関数tailはリストの残りの部分を取り出す。

Prelude> tail [3, 4, 5]
[4,5]
Prelude> tail "Hello"
"ello"

関数myTailを定義する。

myTail :: [a] -> [a]
myTail (_ : xs) = xs

試してみる

myHeadTail.hs

% ghci myHeadTail.hs
*Main> myHead [3, 4, 5]
3
*Main> myTail [3, 4, 5]
[4,5]
*Main> myHead []
*** Exception: myHeadTail.hs:2:1-18: Non-exhaustive patterns in function myHead

*Main> myTail []
*** Exception: myHeadTail.hs:5:1-20: Non-exhaustive patterns in function myTail

空リストに対しては「パターンが存在しない」というエラーが発生する。

全域関数と部分関数

関数とはある集合から別の集合に値を対応づけるものだ。 たとえば逆数を求める関数は2を0.5に4を0.25に対応づける。 逆数関数は0に対して対応する値を持たない。 対応する値のない値がもとの集合にあるような関数は「全域関数ではない部分関数」だ。

headやtailは空リストに対して定義されないので全域関数ではない。 このような関数は慎重に使わないと予期せぬエラーの原因となる。

エラー値

パターンマッチを書かない場合上記のようにNon-exhaustive patternsという エラーメッセージが表示される。 より明示的にエラー値を返すことができる。 定義済みのエラー値としてundefined値と関数errorとがある。

*Main> undefined
*** Exception: Prelude.undefined
*Main> error "Beep! Beep!"
*** Exception: Beep! Beep!

undefinedはコーディングの途中にスタブとして使える。 関数errorを使うと自分の好きなエラーメッセージが表示できる。 関数errorは「起こり得ない場合」に使うことが望ましい。

myHeadのエラーメッセージをユーザーフレンドリーにしてみよう。

myHead (x : _) = x
myHead _ = error "Bonehead!"

関数null

関数nullはリストが空かどうかをチェックする関数だ。

*Main> null [1, 2, 3]
False
*Main> null []
True
*Main> null "hello"
False
*Main> null ""
True

リストに対するパターンマッチを行う。

myNull :: [a] -> Bool
myNull [] = True
myNull (_ : _) = False

これは「空リストなら真、リストに要素をひとつ足したリストなら偽」ということだ。 簡潔にできる。

myNull [] = True
myNull _ = False

「空リストなら真、そうでないなら偽」ということだ。

まとめ

リストは「リストの先頭に要素を足したものまたは空リスト」だ。 リスト型は再帰的に定義される。

空でないリストはx : xsのように作られこの形のまま保存される。 リストはx : xsの形でパターンマッチできる。 [x, y, ..., z]のような表記は構文糖だ。 空リストは[]のように書き[]でパターンマッチできる。

リストを分解して使う関数はパターンマッチを使って書ける。

fun (x : xs) = [xやxsを使った式]
fun [] = [空リストの場合の式]

[]の代わりにワイルドカードが使える。

fun (x : xs) = [xやxsを使った式]
fun _ = [空リストの場合の式]

空リストへのマッチを先にすることもできる。

fun [] = [空リストの場合の式]
fun (x : xs) = [xやxsを使った式]
