型宣言

記法

Haskellでは関数や値の定義の際に型宣言をつけられる。 

[変数名] :: [型]

略記法

同じ型の変数をいくつか定義する場合には略記法がある。

[変数名1], [変数名2], [変数名3] :: [型]

演算子

演算子は丸括弧で囲って関数に変換する。

例

num1 :: Integer
str1 :: String
chr1, chr2, chr3 :: Char
xxcrypt :: Integer -> Integer -> Integer -> Integer
(<!>) :: String -> Integer -> Bool

型宣言をつける意義

Haskellでは型宣言は任意だ。 型宣言がなければコンパイラが型を推論してくれる。 しかし、以下のような理由から、 完全に使い捨てのスクリプト等でなければ型宣言はつけたほうが良い。

  • 自分が何を作っているのかが明確になる
  • バグの多くは型の不一致をともなう
  • 型宣言はドキュメントとして優れる
  • 複雑な型では型推論が利かない場合がある
