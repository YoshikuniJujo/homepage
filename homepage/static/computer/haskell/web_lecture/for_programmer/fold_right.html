(工事中 20%)

はじめに

リストの総和、総積、長さを求める関数を作成した。 これら3つの関数は同じ枠組みを共有している。 二引数関数を使ってリストをひとつの値に変換するという枠組みだ。 このような枠組みを「たたみこみ」と呼ぶ。 共通の枠組みは関数によって抽出できる。 「たたみこみ」には二種類ある。 まずは「右からのたたみこみ」を見てみよう。

それぞれの関数

総和、総積を求める関数の定義を見てみよう。

mySum [] = 0
mySum (x : xs) = x + mySum xs

myProduct [] = 1
myProduct (x : xs) = x * myProduct xs

これらの関数の定義はほとんど同じ形だ。 mySumとmyProductのちがいは0が1になっていることと(+)が(*)になっていることだ。

右たたみこみ

mySum [3, 7, 8, 4]を展開してみよう。

mySum [3, 7, 8, 4]
-> mySum (3 : [7, 8, 4])
-> 3 + mySum [7, 8, 4]
-> 3 + mySum (7 : [8, 4])
-> 3 + (7 + mySum [8, 4])
-> 3 + (7 + mySum (8 : [4]))
-> 3 + (7 + (8 + mySum [4]))
-> 3 + (7 + (8 + mySum (4 : [])))
-> 3 + (7 + (8 + (4 + mySum [])))
-> 3 + (7 + (8 + (4 + 0)))

同様にmyProduct [3, 7, 8, 4]は以下のように展開できる。

3 * (7 * (8 * (4 * 1)))

mySumやmyProductはリストの要素に右結合で演算子を次々と適用していく。 このような操作を「右たたみこみ」と呼ぶ。

置き換え

リスト[3, 7, 8, 4]は以下のようになっている。

3 : (7 : (8 : (4 : [])))

mySumでは[]を0に(:)を(+)に置き換えている。 myProductでも同様で[]を1に(:)を(*)に置き換えている。 []と(:)を何に置き換えるかを変えれば様々な「右たたみこみ」関数が作れる。

右たたみこみを抽象化する

[]の置き換え先と(:)の置き換え先を引数とする右たたみこみを抽象化した関数が作れる。 右たたみこみ関数を作る関数が作れる。 これは関数foldrとしてPreludeモジュールで公開されている。

関数foldrの使いかた

関数mySumをfoldrを使って表す。 関数foldrの第1引数は(:)を置き換える演算子であり第2引数は[]を置き換える値である。 mySumではそれぞれ(+)と0だ。

mySum = foldr (+) 0

関数myProductでは同様にそれぞれ(*)と1を指定する。

myProduct = foldr (*) 1

関数foldrの定義

関数myFoldrを作っる。 mySumとmyProductの定義を参考にする。

mySum [] = 0
mySum (x : xs) = x + mySum xs

myProduct [] = 1
myProduct (x : xs) = x * mySum xs

mySumの0や(+)、myProductの1や(*)を、myFoldrの第1引数と第2引数に置き換える。

myFoldr _ v [] = v
myFoldr op v (x : xs) = x `op` myFoldr op v xs

myFoldrの定義のvを0にopを(+)にmyFoldr op vをmySumに置き換えるとmySumの定義になる。 同様にvを1にopを(*)にmyFoldr op vをmyProductに置き換えるとmyProductの定義になる。

関数foldrの型

第1引数はそれぞれの要素を結合する演算子だ。 第2引数は1番右に来る値、初期値だ。 第3引数はたたみこみの対象となるリストだ。 mySumではそれぞれ以下のような型になる。

(+) :: Integer -> Integer -> Integer
0 :: Integer
xs :: [Integer]

これを一般化して以下のような型が考えられる。

foldr :: (a -> a -> a) -> a -> [a] -> a

2つの型aから型aを作る演算子と型aの値を取り、 型aの値のリストから型aの値を作る関数である。 この型はまだ十分に一般的ではない。 foldrはもっと広い範囲に使える。
