たたみこみ(右)

はじめに

リストの総和、総積、長さを求める関数を作成した。 これらの関数は同じ枠組みを共有している。 2引数関数を使ってリストをひとつの値に変換するという枠組みだ。 このような枠組みを「たたみこみ」と呼ぶ。 共通の枠組みは関数によって抽出できる。 「たたみこみ」には2種類ある。 「右からのたたみこみ」から学ぶ。

それぞれの関数

総和、総積、長さを求める関数の定義を見てみよう。

mySum [] = 0
mySum (x : xs) = x + mySum xs

myProduct [] = 1
myProduct (x : xs) = x * myProduct xs

myLength [] = 0
myLength (_ : xs) = 1 + myLength xs

これらの関数の定義はほとんど同じ形だ。 関数mySumとmyProductのちがいは0が1になっていることと(+)が(*)になっていることだ。 関数mySumとmyLengthのちがいはxが1になっていることだけだ。

右たたみこみ

mySum [3, 7, 8, 4]を展開してみよう。

mySum [3, 7, 8, 4]
-> mySum (3 : [7, 8, 4])
-> 3 + mySum [7, 8, 4]
-> 3 + mySum (7 : [8, 4])
-> 3 + (7 + mySum [8, 4])
-> 3 + (7 + mySum (8 : [4]))
-> 3 + (7 + (8 + mySum [4]))
-> 3 + (7 + (8 + mySum (4 : [])))
-> 3 + (7 + (8 + (4 + mySum [])))
-> 3 + (7 + (8 + (4 + 0)))

同様にmyProduct [3, 7, 8, 4]は以下のように展開できる。

3 * (7 * (8 * (4 * 1)))

myLength [3, 7, 8, 4]は以下のようになる。

1 + (1 + (1 + (1 + 0)))

mySumやmyProductはリストの要素に右結合で演算子を次々と適用していく。 このような操作を「右たたみこみ」と呼ぶ。

置き換え

リスト[3, 7, 8, 4]は以下のようになっている。

3 : (7 : (8 : (4 : [])))

mySumでは[]を0に(:)を(+)に置き換えている。 myProductでも同様で[]を1に(:)を(*)に置き換えている。 []と(:)を何に置き換えるかを変えれば様々な「右たたみこみ」関数が作れる。

関数myLength

関数myLengthでは(:)を置き換える演算子が自明ではない。 以下のようなlen関数を考える。

len :: a -> Int -> Int
len _ l = 1 + l

するとmyLength [3, 7, 8, 4]は以下のようになる。

3 `len` (7 `len` (8 `len` (4 `len` 0)))

このようにするとmyLength関数は[]を0に(:)をlenに置き換えた関数であることがわかる。

右たたみこみを抽象化する

[]の置き換え先と(:)の置き換え先を引数とする右たたみこみを抽象化した関数が作れる。 これは右たたみこみ関数を作る関数となる。 関数foldrとしてPreludeモジュールで公開されている。

関数foldrの使いかた

関数mySumをfoldrを使って表す。 関数foldrの第1引数は(:)を置き換える演算子であり第2引数は[]を置き換える値である。 mySumではそれぞれ(+)と0だ。

mySum = foldr (+) 0

関数myProductでは同様にそれぞれ(*)と1を指定する。

myProduct = foldr (*) 1

関数myLength

関数myLengthでは関数lenを使って以下のように書ける。

myLength = foldr len 0

もっと直接的に定義してみよう。まずはlenを書き換えていく。

len _ l = 1 + l

演算子の部分適用で以下のようになる。

len _ = (1 +)

引数を問わずいつも(1 +)を返す関数なので関数constで書き直せる。

len = const (1 +)

これを使えばmyLengthは以下のように定義できる。

myLength = foldr (const (1 +)) 0

関数foldrの定義

関数myFoldrを作る。 mySumとmyProductの定義を参考にする。

mySum [] = 0
mySum (x : xs) = x + mySum xs

myProduct [] = 1
myProduct (x : xs) = x * mySum xs

mySumの0や(+)、myProductの1や(*)を、myFoldrの第1引数と第2引数に置き換える。

myFoldr _ v [] = v
myFoldr op v (x : xs) = x `op` myFoldr op v xs

myFoldrの定義のvを0にopを(+)にmyFoldr op vをmySumに置き換えるとmySumの定義になる。 同様にvを1にopを(*)にmyFoldr op vをmyProductに置き換えるとmyProductの定義になる。

関数foldrの型

第1引数はそれぞれの要素を結合する演算子だ。 第2引数は1番右に来る値、初期値だ。 第3引数はたたみこみの対象となるリストだ。 返り値はたたみこみの結果得られる値である。

関数mySumからの一般化

mySumではそれぞれ以下のような型になる。

(+) :: Integer -> Integer -> Integer
0 :: Integer
xs :: [Integer]
返り値 :: Integer

foldr :: (Integer -> Integer > Integer) -> Integer -> [Integer] -> Integer

これを一般化して以下のような型が考えられる。

foldr :: (a -> a -> a) -> a -> [a] -> a

2つの型aから型aを作る演算子と型aの値を取り、 型aの値のリストから型aの値を作る関数である。 この型はまだ十分に一般的ではない。 foldrはもっと広い範囲に使える。

関数myLengthからの一般化

myLengthではそれぞれ以下のような型になる。

(const (1 +)) :: a -> Int -> Int
0 :: Int
xs :: [a]
返り値 :: Int

foldr :: (a -> Int -> Int) -> Int -> [a] -> Int

これを一般化する。

foldr :: (a -> b -> b) -> b -> [a] -> b

foldr op v [x1, x2, x3]はx1 `op` (x2 `op` (x3 `op` v))と展開されることを考える。 opの第1引数の型はx1の型と等しく、 第2引数の型はopの返り値の型と等しく、 さらにその型は値vの型と等しいことがわかる。 また全体の返り値の型はopの返り値の型と等しい。 よって上記の型となることがわかる。

まとめ

右たたみこみ関数foldrを学んだ。 関数mySum, myProduct, myLengthなどの関数が持つ「たたみこみ」という枠組み そのものを表現した関数である。 Haskellの持つ「抽象化」の力を理解してもらえただろうか。 さらに学習を進めていくと抽象化の層について実感してもらえることと思う。
