NML(Nano Markup Language): 2. 他の型への変換

Books.hs

はじめに

NMLで表現された本のリストをHaskellのデータ構造Bookに読みこんでみよう。

型Book

本は題名と作者で表現する。

data Book = Book {
title :: String
author :: String }
deriving Show

変換関数

Nml型の値から[Book]型の値への変換関数をつくる。

NMLによる表現

NMLによる本のリストは以下のような表現とする。

<books>
<book>
<title>The Old Man and the Sea</title>
<author>Ernest Hemingway</author>
</book>

<book>
<title>The Catcher in the Rye</title>
<author>J. D. Salinger</author>
</book>
</books>

このサンプルを文字列として定義する。

books1 :: String
books1 = "<books>" ++
"<book>" ++
"<title>The Old Man and the Sea</title>" ++
"<author>Ernest Hemingway</author>" ++
"</book>" ++
"<book>" ++
"<title>The Cather in the Rye<title>" ++
"<author>J. D. Salinger<author>" ++
"</book>" ++
"</books>"

モジュールNmlを使うので

import Nml

をモジュールの先頭に追加しておく。

関数booklistNml

関数booklistNmlは1番外側のタグがbooksであることを確認したうえで なかのbookのリストを変換する。

booklistNml :: Nml -> Maybe [Book]
booklistNml (Node "books" bl) = Just $ mapMaybe book bl
booklistNml _ = Nothing

関数bookはまだ作っていない。Nml -> Maybe Book型の関数だ。 mapMaybeはMaybe型を返す関数をリストの全要素に適用し、Just値だけを集める。 構文エラーや型エラーのチェックのために対話環境で読みこんでみる。 mapMaybeを使うためにData.Maybeをインポートする。 関数bookのスタブも用意する。モジュールの先頭に

import Data.Maybe

を追加し関数bookのスタブを追加する。

book :: Nml -> Maybe Book
book = undefined

関数book

関数bookはtitleタグとauthorタグのそれぞれについてそれ自体の存在と その中身の存在を確認し、存在すればそれらから値をとりだしてBook型にまとめる。 関数getTitle, getAuthorのスタブも作る。

book :: Nml -> Maybe Book
book b@(Node "book" _) = case getTitle b of
Just t -> case getAuthor b of
Just a -> Just $ Book { title = t, author = a }
_ -> Nothing
_ -> Nothing
book _ = Nothing

getTitle, getAuthor :: Nml -> Maybe String
getTitle = undefined
getAuthor = undefined

構文エラーや型エラーはない。

関数getTitle, getAuthor

getTitleとgetAuthorとはタグ名が"title"と"author"で違うだけだ。 このような関数同士は違う部分を引数にしてまとめてしまおう。 関数bookのgetTitleとgetAuthorをそれぞれ getValue "title"とgetValue "author"に書き換える。

... = case getTitle b of
Just t -> case getAuthor b of
...

を

... = case getValue "title" b of
Just t -> case getValue "author" b of
...

とする。スタブも

getTitle, getAuthor :: Nml -> Maybe String
getTitle = undefined
getAuthor = undefined

を

getValue :: String -> Nml -> Maybe String
getValue = undefined

とする。

*Main>

関数getValue

特定のタグを持つ子要素を集めてくれる関数childrenがあれば 関数getValueは

getValue :: String -> Nml -> Maybe String
getValue t b = case children t b of
Node _ [Node v []] : _ -> Just v
_ -> Nothing

と書ける。

children t bの値をパターンマッチして中の値をとりだしている。 パターンNode _ [Node v []] : _は(Node _ [Node v []]) : _ということであり、 Node _ [Node v []]の部分が一番はじめの子要素だ。 子要素のなかには値がひとつ含まれる。 Tree型では葉となる要素がNode v []のように表現される。

関数children

関数childrenは子要素のうち特定のタグを持つ要素だけを集める関数だ。

children :: String -> Nml -> [Nml]
children tg (Node _ cs) = filter ((== tg) . rootLabel) cs

関数booklist

文字列から直接[Book]型の値を入手する関数を作る。

booklist :: String -> Maybe [Book]
booklist s = case nml s of Just bl -> booklistNml bl; _ -> Nothing

モジュールBooks

後で使いやすいようにモジュール名をつける。先頭に

module Books (Books(..), booklist) where

を追加する。
