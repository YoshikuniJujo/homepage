IOMcn型: まとめ

(工事中 60%)

一般の言語では

一般の言語では関数の評価のタイミングで入出力動作を行い、 入力値は関数の返り値としてうけとる。

参照透過性と遅延評価

関数の返り値として入力値を扱うと参照透過ではなくなる。 また関数の評価のタイミングで入出力動作を行うと評価の順序が コードの意味に影響をおよぼしてしまう。

IOを行う機械

機械を組み立てていき最終的な評価結果が「機械」であるようにする。 その「機械」は評価と独立した機構で「実行」されると考えることができる。 機械は入力と出力の型がそれぞれ決められていて

IOMcn a b

のような型となる。 機械に期待される機能をすべて実現するために

(>>>) :: IOMcn a b -> IOMcn b c -> IOMcn a c
arr :: (a -> b) -> IOMcn a b
app :: IOMcn (IOMcn a b, a) b

のみっつのインターフェースがあればいい。
