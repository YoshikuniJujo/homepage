関数xxcrypt

encrypt関数とdecrypt関数は実は同じものだ。

encrypt n e m = m ^ e `mod` n
decrypt n d c = c ^ d `mod` n

ひとつにまとめてしまう。

xxcrypt n ed mc = mc ^ ed `mod` n

鍵を固定した関数

同じ鍵でいろいろなメッセージを暗号化、復号するとき、 いちいち鍵を引数として与えるのは面倒だ。 鍵を固定した関数を作成する。 

encrypt1 m = xxcrypt 138689 13 m
decrypt1 c = xxcrypt 138689 95497 c

試してみる。

*Main> :reload
*Main> encrypt1 4492
47255
*Main> decrypt1 47255
4492

encrypt1やdecrypt1の定義では、それぞれencryptやdecryptの 第1、第2引数だけを指定し、第3引数は指定されていない。 このような場合、以下のように書ける。 

encrypt1' = xxcrypt 138689 13
decrypt1' = xxcrypt 138689 95497

encrypt1'、decrypt1'はそれぞれencrypt、decryptの3つの引数のうち 第1引数と第2引数のみに値を与えることで作成した。 関数の引数のうち一部だけ指定することを部分適用と呼ぶ。 第1引数のみを指定することもできる。 

xxcrypt1 = xxcrypt 138689

これを使ってencrypt1やdecrypt1を定義することもできる。

encrypt1'' = xxcrypt1 13
decrypt1'' = xxcrypt1 95497

展開

encrypt1'' 4492を展開してみる。 

encrypt1'' 4492
==> (xxcrypt1 13) 4492
==> ((xxcrypt 138689) 13) 4492

この展開を逆にたどってみよう。 xxcryptに引数138689を与えるとxxcrypt1という関数を返す。 そのxxcrypt1に引数13を与えるとencrypt1''という関数を返す。 そしてencrypt1''に4492を与えることで数値が返る。

xxcryptという3引数関数に1つ引数を与えてやると、 2引数関数が返り、その関数にまた1つ引数を与えてやると1引数関数が返り、 そしてその関数にまた1つ引数を与えてやると数値が返る。

3引数関数はなかった

xxcryptを今までは3引数関数として扱ってきた。 しかし、実はHaskellに2引数関数や3引数関数はない。 2引数関数や3引数関数のように見えるものは、実は「関数を返す関数」だ。 関数に3つの引数を与えているかのように見える以下のような記述を見てみよう。 

xxcrypt 138689 13 4492
これは本当は以下のような意味である。 
((xxcrypt 138689) 13) 4492

関数適用が左結合なのでこの括弧は省略できる。 結果として上のような表記になる。

まとめ

関数は一部の引数だけを指定できる。 これを部分適用と呼ぶ。 「部分適用」は特別に用意された機能ではない。 「複数の引数を持つ関数」のほうが幻で「関数を返す関数」の連鎖が起きている。
