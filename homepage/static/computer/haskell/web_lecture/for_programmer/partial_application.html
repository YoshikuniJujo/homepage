関数xxcrypt

encrypt関数とdecrypt関数は同じものだ。

encrypt n e m = m ^ e `mod` n
decrypt n d c = c ^ d `mod` n

まとめてしまおう。

xxcrypt n ed mc = mc ^ ed `mod` n

鍵を固定する

同じ鍵でいろいろなメッセージを暗号化、復号する。 いちいち鍵を引数として与えるのは面倒だ。鍵を固定する。

encrypt1 m = xxcrypt 138689 13 m
decrypt1 c = xxcrypt 138689 95497 c

*Main> :reload
*Main> encrypt1 4492
47255
*Main> decrypt1 47255
4492

encrypt1やdecrypt1の定義では、encryptやdecryptの 第1、第2引数だけが指定され第3引数は指定されていない。 以下のように書き直せる。 

encrypt1' = xxcrypt 138689 13
decrypt1' = xxcrypt 138689 95497

encrypt1'、decrypt1'の定義ではencrypt、decryptの3つの引数のうち 第1引数と第2引数のみを与えている。 関数の引数の一部だけ指定することを部分適用と呼ぶ。 第1引数のみを指定することもできる。 

xxcrypt1 = xxcrypt 138689

これを使ってencrypt1やdecrypt1を定義しても良い。

encrypt1'' = xxcrypt1 13
decrypt1'' = xxcrypt1 95497

展開

encrypt1'' 4492
=> (xxcrypt1 13) 4492
=> ((xxcrypt 138689) 13) 4492

逆にたどる。 xxcryptに引数138689を与えるとxxcrypt1という関数になる。 xxcrypt1に引数13を与えるとencrypt1''という関数になる。 encrypt1''に4492を与えると数値になる。

3引数関数に引数を与えると2引数関数が返る。 2引数関数に引数を与えると1引数関数が返る。 1引数関数に引数を与えると数値が返る。

3引数関数は幻だった

xxcryptを3引数関数として扱ってきた。 本当はHaskellに2引数関数や3引数関数はない。 2引数関数や3引数関数のように見えるものは「関数を返す関数」だ。

xxcrypt 138689 13 4492

関数に3つの引数を与えているかのように見える。 明示的に丸括弧をつける。 

((xxcrypt 138689) 13) 4492

関数適用が左結合なのでこの括弧は省略できる。 結果としてうえのようになる。

まとめ

関数は引数の一部だけを指定できる。 この「部分適用」は特別に用意された機能ではない。 「複数の引数を持つ関数」のほうが幻だ。「関数を返す関数」の連鎖があるだけだ。
