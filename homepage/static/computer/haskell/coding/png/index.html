概要

PNGのcodecをいろいろなやりかたで書いていく。 まず一括変換だと大きいファイルをすこしずつ処理していけないということを示す。 次にIOを利用した逐次変換ではファイルの読みこみ等のIOの処理と codecの処理が癒着していて柔軟性がないことを示す。 そこで遅延IOがそれを解決するように見えるが遅延IOには IOの行われるタイミングの恣意性があり危険であることを示す。 それを解決するconduitの威力を見る。
