そんなに本気じゃない暗号化ソフト

(工事中)

動機

テキストファイルとしてバージョン管理したいが、 そんなに公けにしたくないファイルがあるとする。 例えば授業で扱う問題の答えなどはネット上に流したくない。 それでも、問題と一緒に公開リポジトリに置いておいたほうが便利だ。 そんなときに、暗号としての強度は低くても、 簡単に暗号化してバージョン管理ができたら便利だ。

名前

Not Serious Cryptあたりから一応nscという名前にしておく。

仕様等

  • 行単位で暗号化する
    • つまり改行は改行のままにする(\nのみに対応)
    • バージョン管理ソフトとの相性が良いので
  • 結果はBase64で表現する
    • ASCIIテキストのほうが表示等で扱いやすいため
  • 暗号化アルゴリズムはAES-CBCとする
    • 一番メジャーなように思うので
    • 深い意味はない
  • IVはランダム値として暗号化された行の先頭に付ける
    • いろいろと良い面がある
      • 暗号化や複号化が行単位で並列化できる等
      • しないけど
    • 本気でないので標準の(暗号化用でない)ランダムを使う
  • 公開するほどではないので今はHackageで公開しない
  • 必要なパッケージが明らかになるのでcabal化はしておく
