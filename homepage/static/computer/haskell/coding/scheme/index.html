ただ、実装や動作が汚なくなりそうならば「きれい」なほうをとるかもしれない。

ひな形的な

とりあえずREPLが動かないと面白くないのでひな形的なコードを作成する。 これを拡充していこう。 ただし、途中で根本的な設計の変更が必要になったならコードを捨てるかもしれない。
