バイナリ解析器: ListLikeモジュールの作成

(工事中 50%)

動機

バイナリ解析器は、文字やその他のリストにも使え、ByteStringにも使えるようにしたい。 そのためには「リストのようなもの」クラスが必要になる。

TypeFamilies拡張

ByteStringをリストのようなものと考えたときに、 その要素はWord8と考えることができる。 よって、ByteStringをListLikeクラスのインスタンスにしたとき、 要素となる値の型はWord8である。 しかし、別の何かをリストのようなものと考えたときには、 また別の型の値がその要素となるだろう。 このように、クラスのインスタンスとなる型から、 他の型を導き出す必要があることがしばしばある。 このようなときにはTypeFamilies拡張を使い以下のように書けば良い。

クラス宣言

class ListLike a where
	type Element a

インスタンス宣言

instance ListLike ByteString where
	type Element ByteString = Word8
instance ListLike [a] where
	type Element [a] = a

リストのようなものに必要な性質

必要な性質(クラス関数)

「リストのようなもの」に必要な性質は、 読み込みに必要な性質と書き出しに必要な性質とに分けられる。

読み込みに必要な性質は、 「頭を取ることができる」「残りを取ることができる」「空かどうか調べられる」である。 これは以下のような関数でこの3つを同時に行うことができる。 

uncons :: a -> Maybe (Element a, a)

書き出しに必要な性質は、 「頭と[リスト的なもの]とをつなげて新しい[リスト的なもの]を作る」 「空の[リスト的なもの]」である。 これはそれぞれ以下のような関数で表現することができる。 

cons :: Element a -> a -> a
empty :: a

クラス宣言

ここまででListLikeクラスは以下のようになる。

class ListLike a where
	type Element a
	empty :: a
	cons :: Element a -> a -> a
	uncons :: a -> Maybe (Element a, a)

splitAtの実行効率について

splitAtを上述のクラス関数を使って実装する

与えられた「リスト的なもの」を位置を指定して2分割する関数splitAtを定義する。 以下のようになる。 

splitAt :: ListLike a => Integer -> a -> (a, a)
splitAt n xs
	| n <= 0 = (empty, xs)
	| otherwise = case uncons xs of
		Just (h, t) -> let
			(t1, t2) = splitAt (n - 1) t in
			(cons h t1, t2)
		_ -> (empty, empty)

ByteStringにおける実行効率

参照: Data.ByteString.splitAt

ByteStringでは、バイト列におけるいくつかの演算を 文字のリストであるStringよりも効率的に行うようになっている。 たとえばData.ByteString.splitAtはO(1)つまり定数時間で実行可能である。 しかし、上で定義したListLike.splitAtでは、1バイトずつuncons, consしていくので 実行効率はO(n)となってしまう。 これだと、ByteStringを使う意味が薄れてしまう。

これはsplitAtをListLikeのクラス関数にすることで解決する。 それぞれのインスタンスごとに独自のsplitAtを持つことができる。

splitAtの実行効率が改善しない型の場合

上で定義したままのsplitAtを使ったとしても実行効率に影響が出ない型も多い。 そのような場合、何度も何度も同じsplitAtの定義を書くというのは非効率である。

デフォルトの定義という解決策

クラス関数としたうえで、それにデフォルトの定義を与えるという方法がある。 こうすれば効率等に問題がない型ではデフォルトの定義を使い、 何らかの問題がある場合にだけクラス関数を上書きするというやりかたができる。 

class ListLike a where
	type Element a
	empty :: a
	cons :: Element a -> a -> a
	uncons :: a -> Maybe (Element a, a)
	splitAt :: Integer -> a -> (a, a)
	splitAt n xs
		| n <= 0 = (empty, xs)
		| otherwise = case uncons xs of
			Just (h, t) -> let
				(t1, t2) = splitAt (n - 1) t in
				(cons h t1, t2)
			_ -> (empty, empty)
