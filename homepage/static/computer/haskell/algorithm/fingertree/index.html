Finger Trees: A Simple General-purpose Data Structure

はじめに

状態変化をつかわずに いろんなことが償却定数時間や対数時間でできる 並びを表現したデータ構造がフィンガーツリーだ。 HaskellのモジュールData.SequenceのSeqに使われている。 まずは自分で1から作ってみてそのあとでData.Sequenceのコードを見てみよう。

動機

フィンガーツリーについて調べようと思った直接的な動機は Deflate圧縮/展開にキュー的な並びが必要だからだ。 もちろん配列とIOを使うことも可能だが 「状態変化」なしに実装するのがHaskellerとしてはスマートかと考えた。 Deflate圧縮で使う並びは両端への変更とランダムアクセスが 効率的に行える必要がある。 フィンガーツリーならばどうにか使えるかと考えた。 配列の状態変化を使った方法と比較して大きな効率の低下がなければよしとする。
