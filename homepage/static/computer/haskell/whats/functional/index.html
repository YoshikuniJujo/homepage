はじめに

「関数型プログラミング」とか「関数型言語」という言葉が流行りだしている。 しかしこれらの言葉は定義がはっきりしていない。 人によって指し示すものがまちまちなように思う。 僕個人の考えをまとめてみようかと思う。

なぜ関数プログラミングは重要か

「関数型...」を構成する要素

「関数型プログラミング」というのは定義がはっきりしないが、 「ラムダ計算」あたりの研究の成果を応用したプログラミングの手法というところだ。 以下のような特徴がある。

  • 第1級関数
  • 参照透過性
  • 遅延性
  • 静的型付け
  • 評価と実行の分離

遅延性や静的型付けに関してはそれを「関数型」の特徴に含めるかどうかで 異論があるかもしれない。 これらも「関数型」と関連した分野として研究されている。 いずれにしてもここでは「何が関数型か」にはとらわれずに これらの特徴について何がうれしいかを説明していくことにする。

それXX言語でもできるよ

関数型言語でこれができる、あれができると言ったところで、 モダンな言語では同じようなことができる。 だから関数型言語を学ぶ意味がないかと言うとそうではない。 実のところモダンな言語で同じようなことができるというのは 関数型言語からその特徴の一部をとりこんでいるからだ。

関数型の考えかたを理解し身につけることは最近のプログラミングというものを 知るうえで必須だ。 プログラミングの技法や考えかたの進化は「関数型」の分野で起こっている。

最終的にすべては機械語に翻訳されることを考えれば「関数型プログラミング」は どの言語でもできるとも言える。 それはバイナリエディタを使えば絵も描けるし作曲もできるというのと同じことだ。 お絵描きソフトには作曲機能はないがその代わり絵を描くという機能においては バイナリエディタよりもはるかに効率的だ。 同じように「関数型」のパラダイムでも「手続き型」のパラダイムでもコーディングが 可能なXX言語よりも「手続き型」のパラダイムを捨てた純粋関数型言語のほうが 関数型のコーディングを簡潔に書くことができる。

純粋手続き型言語として機械語があり純粋関数型言語としてHaskellがある。 Haskellは非常に実用性があり開発効率を格段に向上させるものだ。 しかしそれはHaskellを十分に身につけるまで実感するのは難しい。 最終的に手続き型パラダイムを選択するとしても 関数型のパラダイムを1度身につけておくことは価値あることだ。 そして、実のところ関数型プログラミングは簡単だ。 なのでコストパフォーマンスから考えて関数型言語を学ばない理由はない。
