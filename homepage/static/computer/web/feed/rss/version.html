RSS:バージョンの分裂

RDF構文

バージョンの分裂の核心としてRDF構文の使用の有無がある。 RDF構文を採用することで多くの機能を提供できるが、その反面複雑になってしまう。 逆にRDF構文を捨てることによって、コンテンツ配信以外の多くの機能を失うが、 簡単にすることができる。 といったところか。

RSS 0.9

RDF site summary。 RSSの最初のバージョン。 RDF構文を使用している。

RSS 0.91

rich site summary。 RDF構文を捨てた。 その他「コンテンツ配信」機能に特化して様々な拡張がされている。

RSS 1.0

RDF構文を再び採用。 中核機能以外は拡張モジュールとした。 形としては0.9の後継となる。

RSS 2.0

really simple syndication。 0.91の後継。 RDF構文は使用しない。
