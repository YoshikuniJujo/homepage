8. タグ

8.1

タグはクラスとそのクラス内での番号で指定する。 クラスは以下のうちのどれかである。

  • universal
  • application
  • private
  • context-specific

8.2

番号は非負整数で十進数表記で指定する。

8.3

使用者によって付加されるタグの制約は31.2で定める。

NOTE - Subclause 31.2 includes the restriction that users of this notation are not allowed to explicitly specify universal class tags in their ASN.1 specifications. There is no formal difference between use of tags from the other three classes. Where application class tags are employed, a private or context-specific class tag could generally be applied instead, as a matter of user choice and style. The presence of the three classes is largely for historical reaseons, but guidance is given in G.2.12 on the way in which the classes are usually employed.

8.5

タグの正統な順序を必要とする符号化規則がある。 一意性を提供するためにタグの正統な順序を8.6に定める。

8.6

タグの正統な順序はそれぞれの型の最外タグを見て以下の規則によって定める。

  • a) 要素や選択肢は、 universalクラスタグが最初に来て、applicationクラスタグ、 context-specificクラスタグ、privateクラスタグと続く。
  • b) それぞれのタグのクラス内では、 これらの要素または選択肢はタグ番号の昇順で配置される。
