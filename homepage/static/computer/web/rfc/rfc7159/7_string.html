RFC 7159 7. 文字列

文字列の表現はC系のプログラミング言語の慣習と似ている。 文字列の始めと終わりは引用符だ。 それらの引用符のなかに以下のエスケープが必要な文字以外の すべてのユニコード文字が置ける。 引用符、逆斜線、コントロール文字(U+0000からU+001F)。

どの文字もエスケープできる。 もし文字が基本多言語平面(U+0000からU+FFFF)にあれば、 以下の6文字の並びで表現できる。 逆斜線、小文字のu、 その文字のコードポイントを表す4桁の16進数字。 16進文字のAからFは大文字でも小文字でも良い。 たとえば、逆斜線のみを含む文字列は"\u005C"と表せる。

代替手段として、よく使われる文字については、2文字でのエスケープ表現がある。 たとえば、逆斜線のみを含む文字列は"\\"と表せる。

基本多言語平面に含まれない拡張文字をエスケープするには、 UTF-16のサロゲートペアを符号化する12文字の並びで表現する。 たとえば、ト音記号(U+1D11E)のみを含む文字列は"\uD834\uDD1E"と表せる。

string = quotation-mark *char quotation-mark

char = unescaped /
%x22 / ; " 引用符 U+0022
%x5C / ; \ 逆斜線 U+005C
%x2F / ; / 斜線 U+002F
%x62 / ; b バックスペース U+0008
%x66 / ; f 改ページ U+000C
%x6E / ; n 行送り U+000A
%x72 / ; r 復帰 U+000D
%x74 / ; t タブ U+0009
%x75 4HEXDIG ) ; uXXXX U+ XXXX

escape = %x5C ; \

quotation-mark = %x22 ; "

unescaped = %x20-21 / %x23-5B / %x5d-10FFFF
