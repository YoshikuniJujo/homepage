RFC 7159 2. 文法

(工事中 0%)

JSON文書はトークンの並びだ。 トークンは6つの構文文字、文字列、数値、3つのリテラル名から成る。

JSON文書は直列化された値だ。 以前のJSONの仕様ではJSON文書をオブジェクトまたは配列と規定していたものがある ことに注意する。 [以下1文、あとで訳す] Implementations that generate only objects or arrays where a JSON text is called for will be interoperable in the sense that all implementations will accept these as conforming JSON texts.

JSON-text = ws value ws

以下に6つの構文文字を示す:

begin-array = ws %x5B ws; [ 左角括弧

begin-object = ws %x7B ws; { 左波括弧

end-array = ws %x5D ws; ] 右角括弧

end-object = ws %x7D ws; } 右波括弧

name-separator = ws %x3A ws; : colon

value-separator = ws %x2C ws; , comma

意味のない空白は6つの構文文字の前にも後ろにもつけられる。

ws = *(
%x20 / ; スペース
%x09 / ; 水平タブ
%x0A / ; 行送りまたは改行
%x0D ) ; 復帰
